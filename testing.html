Introduction

Selenium is an open source Browser based Test Automation Software. It is a robust set of suite of tools which supports rapid development of test automation for web-based applications. It has support for many other languages than just HTML 5.

The Selenium IDE is very user-friendly. It has execution commands, a record button and Web based test runner option. It also has reference and log tabs where user can see details about the command currently being run and progress of the test respectively.

Genral Commands

  • cliking a link- click or clickAndWait commands
  • entering values- type command
  • selecting options from a drop-down listbox- select command
  • click checkboxes or radio buttons- click command

How did we test?

First of all, Selenium IDE for Mozilla Firefox was installed. As soon as the localhost was fired up, the Selenium IDE started recording the sequence user's actions and clicks in the test suite. As a user, the team went through all use cases after which the test suite was run to start automated testing.

Errors & Debugging

  • timeout - The actual application was receiving a session timeout because the Sentiment Analysis API took too long to respond.
  • The team fixed this error by manually colour coding the tweets if the call to Sentiment Analysis API takes too long.
